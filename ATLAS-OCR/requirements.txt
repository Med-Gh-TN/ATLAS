# CRITICAL FIX: onnxruntime-gpu → onnxruntime
# onnxruntime-gpu ships WITHOUT the CPUExecutionProvider C++ kernels in its
# default wheel. When FASTEMBED_PROVIDER=CPUExecutionProvider is set, the
# GPU build falls through to a "best-effort" provider search that either:
#   (a) tries to load libcuda.so and crashes on machines without CUDA runtime
#   (b) silently degrades to a fallback that is missing the Slice kernel
#       required by Jina-ColBERT-v2's attention head → "out-of-bounds" crash
# The standard `onnxruntime` wheel contains the full CPU kernel library.
